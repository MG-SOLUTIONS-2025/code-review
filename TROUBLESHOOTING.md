# Troubleshooting

## Services Won't Start

### Ollama: "connection refused" or unhealthy

```bash
docker logs ollama
```

**Model not downloaded yet:**
Ollama auto-pulls on first request, but this takes time. Wait 2-5 minutes, then:
```bash
docker exec ollama ollama list
```
If empty, pull manually:
```bash
docker exec ollama ollama pull qwen2.5-coder:32b
```

**CPU mode — healthcheck fails:**
The default healthcheck expects GPU. The `docker-compose.override.yml` relaxes this.
Verify it's being used:
```bash
docker compose config | grep -A3 "ollama"
```

### vLLM: "out of memory"

```bash
docker logs vllm
```

- Reduce `VLLM_TENSOR_PARALLEL_SIZE` if you have fewer GPUs
- Use a smaller model: `VLLM_MODEL=Qwen/Qwen2.5-Coder-7B-Instruct`
- Check GPU memory: `nvidia-smi`

### DefectDojo: initializer fails

```bash
docker logs defectdojo-initializer
```

Common causes:
- `DEFECTDOJO_SECRET_KEY` is still the default — generate a real one: `openssl rand -hex 32`
- `DEFECTDOJO_ADMIN_PASSWORD` too short — must be 8+ characters
- Postgres not ready — restart: `docker compose restart defectdojo-initializer`

### Nginx: "No such file: /etc/nginx/.htpasswd"

The `.htpasswd` file is generated by `scripts/setup.sh`. Create it manually:
```bash
mkdir -p config/nginx
htpasswd -bc config/nginx/.htpasswd admin changeme
docker compose restart nginx
```

Or install apache2-utils first: `apt install apache2-utils` / `brew install httpd`

## Reviews Not Triggering

### No review comment appears on MR/PR

1. **Check gateway logs:**
   ```bash
   docker logs api-gateway --tail 50
   ```

2. **Verify webhook is configured:**
   - GitLab: Project > Settings > Webhooks — check for recent deliveries
   - Gitea: Repo > Settings > Webhooks — check delivery history

3. **Test the API directly:**
   ```bash
   curl -X POST http://localhost:8000/api/reviews/run \
     -H "Content-Type: application/json" \
     -H "Authorization: Bearer $GATEWAY_API_TOKEN" \
     -d '{"project_id": "group/project", "mr_id": 42}'
   ```

4. **Check token permissions:**
   - GitLab: `GIT_TOKEN` needs `api` scope, Reporter role
   - Gitea: `GIT_TOKEN` needs `repo` + `write:issue` scope

5. **Already reviewed?** Reviews are skipped if the HEAD SHA hasn't changed.
   Use `"force": true` to re-review:
   ```bash
   curl -X POST http://localhost:8000/api/reviews/run \
     -H "Content-Type: application/json" \
     -d '{"project_id": "group/project", "mr_id": 42, "force": true}'
   ```

### Review comment is empty or truncated

- **LLM timed out:** Check `docker logs ollama` for timeout errors
- **Diff too large:** Reduce `MAX_PATCH_CHARS` (default 8000) or the diff will be truncated
- **Token budget too small:** Increase `TOKEN_BUDGET_REVIEW` (default 24000)

### "Rate limited" (429) when triggering reviews

`/api/reviews/run` is limited to 5 requests/minute per IP. Wait and retry, or adjust the limit in `api-gateway/src/gateway/routes/reviews.py`.

## Dashboard Issues

### Dashboard shows "Failed to load reviews"

- Check API gateway is running: `curl http://localhost:8000/api/health`
- Check CORS: `CORS_ALLOWED_ORIGINS` must include the dashboard URL (default: `http://localhost:5173,http://localhost:3102`)
- Check nginx: `docker logs nginx`

### Settings page crashes on save

This was a known bug (spreading undefined config objects). If you hit this, pull the latest code.

## DefectDojo Integration

### "Failed to fetch findings" (502)

- `DEFECTDOJO_API_TOKEN` not set or expired — regenerate in DefectDojo UI
- DefectDojo service down: `docker logs defectdojo-web`
- Wrong URL: the internal URL is `http://defectdojo-web:8081` (port 8081, not 8080)

### CI upload fails

- Verify `DEFECTDOJO_ENGAGEMENT_ID` is correct (check URL in DefectDojo UI)
- Verify `DEFECTDOJO_URL` is reachable from CI runner
- Check file size — max 50 MB per upload

## Performance

### Reviews take >2 minutes

| Bottleneck | Fix |
|-----------|-----|
| CPU inference | Use GPU or switch to 7B model |
| Large diffs | Reduce `MAX_PATCH_CHARS` |
| Many files | Increase `REVIEW_BATCH_SIZE` (default 6) |
| LLM overloaded | Reduce `LLM_CONCURRENCY_LIMIT` |

### High memory usage

```bash
docker stats
```

- Ollama with 32B model uses ~20 GB VRAM
- DefectDojo + Postgres can use 1-2 GB RAM
- API Gateway is limited to 512 MB (set in docker-compose.yml)

## Checking Logs

```bash
# All services
docker compose logs --tail 50

# Specific service
docker logs api-gateway --tail 100 -f

# Filter for errors
docker logs api-gateway 2>&1 | grep -i error
```

## Resetting Data

**Reset DefectDojo (delete all findings):**
```bash
docker compose down
docker volume rm code-review_defectdojo-db-data
docker compose up -d
```

**Reset Ollama (re-download model):**
```bash
docker compose down
docker volume rm code-review_ollama-data
docker compose up -d
```

**Reset everything:**
```bash
docker compose down -v
docker compose up -d
```
